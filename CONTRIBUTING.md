# Contributing

Thank you for your interest in contributing to `bessa_patterns.ts`!

## Development Setup

**Prerequisites:** Node.js ≥ 18, npm ≥ 9

```bash
git clone https://github.com/mpbarbosa/bessa_patterns.ts.git
cd bessa_patterns.ts
npm install
```

Verify everything works:

```bash
npm run lint    # ESLint with TypeScript rules
npm test        # Jest (46 tests)
npm run build   # tsc → dist/
```

## Project Structure

```
src/           TypeScript source — one file per pattern
test/          Jest tests — mirrors src/ structure
docs/          Markdown documentation
dist/          Compiled output (git-ignored, generated by npm run build)
scripts/       Utility scripts (deploy.sh)
```

## Adding a New Pattern

1. Create `src/<PatternName>.ts` with a concrete, directly instantiable class
1. Export it from `src/index.ts`
1. Create `test/<PatternName>.test.ts` with full coverage
1. Add API documentation to `docs/<PATTERN_NAME>_API.md`
1. Update `docs/ROADMAP.md` to mark the item complete
1. Bump the patch/minor version in `package.json` and `.workflow-config.yaml`

## Code Conventions

- **TypeScript strict mode** — `strict: true` in `tsconfig.json`; no `any`
- **Generics over `any`** — use `unknown` when the type cannot be fixed
- **Concrete classes** — instantiable directly; avoid abstract unless genuinely needed
- **Error isolation** — `try/catch` per observer/callback so one failure doesn't block others
- **Immutable collections** — use spread/filter rather than mutating arrays in place
- **No runtime dependencies** — the library itself has no production `dependencies`

## Pull Request Process

1. Fork the repository and create a feature branch (`feat/strategy-pattern`)
1. Implement your changes following the conventions above
1. Ensure `npm run lint`, `npm test`, and `npm run build` all pass
1. Open a PR against `main` with a clear description of what was added/changed
1. CI will run automatically (Node 18 + 20, lint + test + build)

## Commit Style

Use [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add Strategy pattern
fix: correct unsubscribe return value in ObserverSubject
docs: add DualObserverSubject API reference
chore: bump version to 0.11.0-alpha
```

## Running Tests

```bash
npm test                    # all tests
npm run test:watch          # watch mode
npm run test:coverage       # with coverage report
```

## Reporting Issues

Open an issue at <https://github.com/mpbarbosa/bessa_patterns.ts/issues> with:

- A minimal reproduction case
- Expected vs. actual behaviour
- Node.js and npm versions (`node -v && npm -v`)
